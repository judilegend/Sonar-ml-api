## Quick Start

### Option A ‚Äî Local development (no Docker)

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Train the model (generates models/sonar_model.pkl)
python ml/train.py

# 3. Run the API
uvicorn app.main:app --reload

# 4. Open the interactive docs
# http://localhost:8000/docs
```

### Option B ‚Äî Docker Compose

```bash
# Build and start the API
docker compose up --build

# Train the model inside Docker (if needed)
docker compose run --rm trainer

# The API is available at http://localhost:8000
```

## Endpoints

| Method | URL              | Description                 |
| ------ | ---------------- | --------------------------- |
| GET    | `/health`        | API and model health status |
| GET    | `/model/info`    | Loaded model metadata       |
| POST   | `/predict`       | Single signal prediction    |
| POST   | `/predict/batch` | Batch predictions           |
| GET    | `/docs`          | Swagger UI documentation    |

## CI/CD Pipeline

The `.github/workflows/ci_cd.yml` workflow defines 3 jobs:

1. **Test** ‚Äî lint with flake8 and run pytest (on every push and PR)
2. **Build** ‚Äî build the multi-stage Docker image (on push to main)
3. **Publish** ‚Äî push the image to Docker Hub (on push to main)

### Configure GitHub Secrets

In your repository, go to Settings ‚Üí Secrets and variables ‚Üí Actions:

- `DOCKERHUB_USERNAME` ‚Üí your Docker Hub username
- `DOCKERHUB_TOKEN` ‚Üí [create one here](https://hub.docker.com/settings/security)
